Overview
6R전략 담당 구성원 AI비서봇 생성 가이드
각자의 AI 비서봇을 직접 만들고 운영한다. 이것은 시작일 뿐 — 봇끼리 연결하고, 더 많은 서비스와 붙여나가는 AI 개인비서 플랫폼의 초기 형태다.
🚀 이건 단순한 챗봇이 아니다
오늘 만드는 텔레그램 봇은 AI 개인비서 플랫폼의 첫 번째 레이어다.
봇끼리 연결되면 팀 단위 AI 워크플로우가 되고, 여기에 캘린더·노션·슬랙·이메일을 붙이면 실제 업무 자동화 에이전트가 된다.

지금 이 경험이 그 시작점이다. 직접 만들어야 이해하고, 이해해야 확장할 수 있다.
현재: 개인 AI 비서봇 → 봇 간 연결 (팀 워크플로우) → 외부 서비스 연동 → 업무 자동화 에이전트

🤖 Dify 두뇌에게 내 역할·말투를 부여. 대화 맥락 기억. 시스템 프롬프트 관리 AI가 내가 누구인지 모름, 매번 처음부터 무료

Step 1 / 6
📱 텔레그램 봇 생성
BotFather에서 내 봇 이름을 만들고 Token(봇의 열쇠)을 받는다. 약 5분.
🔑
Bot Token이란? 텔레그램이 "이게 진짜 네 봇"임을 확인하는 비밀 문자열. 나중에 Cloudflare 환경변수에 입력한다. 절대 타인과 공유 금지.
🪪
텔레그램 chat_id(숫자 ID)가 왜 필요한가? 텔레그램은 모든 사용자에게 고유한 숫자 ID를 부여한다. 봇 간 메시지를 보낼 때 "권오혁 TL에게 전달해줘"를 처리하려면 그 사람의 chat_id가 필요하다. 내 chat_id = 다른 봇이 나를 찾는 주소다.

Step 4 / 6
⚡ Cloudflare Worker 배포
봇의 실제 서버. AI가 짜준 코드를 붙여넣으면 즉시 URL이 생긴다. 이 URL이 내 봇의 주소다.
⚡
Cloudflare에서 뭘 하는 건가? 텔레그램↔Dify는 직접 연결이 안 된다. Cloudflare Workers가 중간에서 24시간 연결해준다. 코드를 올리면 [이름].workers.dev URL이 생기고, 이 URL을 텔레그램에 등록하면 봇이 살아난다.
📦
KV(Key-Value Storage)가 뭔가? Cloudflare 안에 있는 간단한 저장소다. 예: "권오혁" → "987654321" 처럼 이름과 chat_id를 저장해두면, 봇이 "권오혁에게 전달해줘"라는 말을 들었을 때 KV에서 ID를 꺼내 메시지를 전달할 수 있다.
⚠️
이 단계에서 환경변수 4개를 모두 입력해야 봇이 작동한다. Step 1~3에서 받은 값들을 준비하자.

📋 예시 코드 — 이동연(A)봇 worker.js
실제 운영 중인 이동연(A)봇 코드. 이름·역할 부분만 바꾸면 본인 봇이 된다. 복사 후 Cloudflare에 붙여넣기.
✅
이 코드는 실제 검증된 코드다. API Key는 코드에 쓰지 않는다. 모두 Cloudflare 환경변수(Settings → Variables)에서 관리.
worker.js — 이동연(A)봇 전체 코드 (복사 후 이름·역할 수정)
// ══════════════════════════════════════════════════
// 이동연(A)봇 worker.js — Cloudflare Workers
// 텔레그램 Webhook → Dify Agent → LLM 답변 반환
// 봇 간 라우팅: "OOO에게 ~해줘" 패턴 처리
// ══════════════════════════════════════════════════

export default {
  async fetch(request, env) {
    // GET 요청 무시 (Webhook은 POST만 사용)
    if (request.method !== 'POST') {
      return new Response('Bot is running');
    }

    let body;
    try {
      body = await request.json();
    } catch {
      return new Response('OK');
    }

    const message = body.message;
    if (!message) return new Response('OK');

    const chatId = message.chat.id.toString();
    const text = message.text || '';

    // ① /관리자등록 — 처음 한 번 실행해서 내 chat_id를 KV에 저장
    if (text === '/관리자등록') {
      await env.BOT_KV.put('admin_id', chatId);
      await sendMsg(chatId, '✅ 이동연봇 관리자 등록 완료.\n이제 명령을 내릴 수 있습니다.', env);
      return new Response('OK');
    }

    // ② 권한 확인 — 등록된 관리자만 처리
    const adminId = await env.BOT_KV.get('admin_id');
    if (!adminId || chatId !== adminId) {
      return new Response('OK'); // 등록되지 않은 사용자 무시
    }

    // ③ 파일(문서) 처리 — "요약" 키워드와 함께 파일 전송 시
    if (message.document) {
      const caption = message.caption || '';
      if (caption.includes('요약')) {
        await sendMsg(chatId, '📄 파일 받았습니다. 요약 중...', env);
        const fileText = await getFileText(message.document, env);
        if (fileText) {
          const reply = await callDify(
            `다음 문서를 핵심만 3~5줄로 요약해줘:\n\n${fileText}`, chatId, env
          );
          await sendMsg(chatId, reply, env);
        } else {
          await sendMsg(chatId, '❌ 파일을 읽을 수 없습니다. 텍스트로 내용을 붙여넣어 주세요.', env);
        }
      } else {
        await sendMsg(chatId, '📎 파일을 받았습니다. "요약"이라고 캡션을 달아서 다시 보내주세요.', env);
      }
      return new Response('OK');
    }

    // ④ "OOO에게 ~해줘" 패턴 — 다른 팀원에게 메시지 전달
    // KV에 저장된 팀원 chat_id로 텔레그램 메시지 발송
    const teamMembers = [
      '권오혁', '구정모', '김민아', '황성욱',
      '이기두', '위예슬', '홍석윤', '성봉구', '염성진'
    ];

    for (const member of teamMembers) {
      if (text.includes(`${member}에게`) || text.includes(`${member}한테`)) {
        const targetId = await env.BOT_KV.get(`chatid_${member}`);
        if (targetId) {
          const fwdMsg = `📩 [이동연 TL 요청]\n${text}\n\n— 이동연봇이 전달함`;
          await sendMsg(targetId, fwdMsg, env);
          await sendMsg(chatId, `✅ ${member}님께 메시지를 전달했습니다.`, env);
        } else {
          await sendMsg(chatId, `⚠️ ${member}님의 봇이 아직 연결되지 않았습니다.`, env);
        }
        return new Response('OK');
      }
    }

    // ⑤ 일반 질문 → Dify Agent 호출
    const reply = await callDify(text, chatId, env);
    await sendMsg(chatId, reply, env);
    return new Response('OK');
  }
};

// ── Dify API 호출 (대화 맥락 유지) ──────────────────
async function callDify(userMessage, chatId, env) {
  try {
    // KV에서 이전 conversation_id 가져오기 (대화 맥락 유지)
    const convId = await env.BOT_KV.get(`conv_${chatId}`);

    const payload = {
      inputs: {},
      query: userMessage,
      response_mode: 'blocking',
      user: `telegram_${chatId}`,
    };
    if (convId) payload.conversation_id = convId;

    const res = await fetch(
      `${env.DIFY_API_URL}/chat-messages`,
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${env.DIFY_API_KEY}`,
          'Content-Type': 'application/json',
        },
        body: JSON.stringify(payload),
      }
    );

    if (!res.ok) {
      const errText = await res.text();
      console.error('Dify error:', errText);
      return '⚠️ AI 응답 오류가 발생했습니다. 잠시 후 다시 시도해주세요.';
    }

    const data = await res.json();

    // 새 conversation_id를 KV에 저장 (다음 대화에서 맥락 유지)
    if (data.conversation_id) {
      await env.BOT_KV.put(`conv_${chatId}`, data.conversation_id);
    }

    return data.answer || '응답을 받지 못했습니다.';
  } catch (e) {
    console.error('callDify error:', e);
    return '⚠️ 서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요.';
  }
}

// ── 텔레그램 메시지 전송 ─────────────────────────────
async function sendMsg(chatId, text, env) {
  await fetch(
    `https://api.telegram.org/bot${env.TELEGRAM_BOT_TOKEN}/sendMessage`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        chat_id: chatId,
        text: text,
        parse_mode: 'HTML', // <b>굵게</b> 등 HTML 태그 허용
      }),
    }
  );
}

// ── 파일 텍스트 추출 (텍스트 파일 한정) ─────────────
async function getFileText(doc, env) {
  try {
    // 텔레그램에서 파일 경로 획득
    const fileRes = await fetch(
      `https://api.telegram.org/bot${env.TELEGRAM_BOT_TOKEN}/getFile?file_id=${doc.file_id}`
    );
    const fileData = await fileRes.json();
    const filePath = fileData.result?.file_path;
    if (!filePath) return null;

    // 파일 다운로드
    const fileUrl = `https://api.telegram.org/file/bot${env.TELEGRAM_BOT_TOKEN}/${filePath}`;
    const fileContent = await fetch(fileUrl);
    const buffer = await fileContent.arrayBuffer();

    // 텍스트 파일만 처리 (PDF 파싱은 외부 서비스 필요)
    const mimeType = doc.mime_type || '';
    if (mimeType.includes('text') || mimeType.includes('plain')) {
      return new TextDecoder('utf-8').decode(buffer);
    }
    return null; // PDF·PPTX는 현재 미지원 → 텍스트 붙여넣기 안내
  } catch {
    return null;
  }
}

Troubleshooting
❓ 막혔을 때 (FAQ)
실습 중 막히면 여기서 먼저 확인. 해결 안 되면 Cloudflare Logs에서 에러를 복사해서 Claude/ChatGPT에게 "이 에러 고쳐줘"라고 붙여넣자.
Q

봇에게 말했는데 아무 반응이 없어요

⌄
체크 1Webhook 등록했나? → setWebhook URL을 브라우저에 다시 입력
체크 2/관리자등록 전송했나? → 처음 한 번은 반드시 해야 함
체크 3환경변수 4개 모두 입력됐나? → Worker → Settings → Variables 확인
체크 4KV Namespace BOT_KV가 Worker에 바인딩됐나? → Settings → Bindings 확인
체크 5Cloudflare → Worker → Logs 탭에서 에러 메시지 확인
Q

Dify에서 LLM 모델이 안 보여요

⌄
원인모델 공급자에 API Key 미입력 또는 잘못 입력
해결Dify → 우측상단 아바타 → 설정 → 모델 공급자 → 해당 LLM → API Key 재입력
Q

Cloudflare에서 500 에러 / 오류 발생

⌄
1Worker → Logs 탭 → 에러 메시지 전체 복사
2Claude/ChatGPT에 "worker.js 코드와 이 에러를 보여줄게, 고쳐줘" 요청
3환경변수명 오타 확인 — 대소문자·언더바 주의 (TELEGRAM_BOT_TOKEN 정확히)
4BOT_KV 바인딩이 코드의 env.BOT_KV와 변수명이 일치하는지 확인
Q

BotFather에서 username이 이미 사용 중이래요

⌄
원인전 세계에서 유일한 username이어야 함
해결이름 뒤에 숫자·팀명 추가 예: hwang_6r_bot, sgwook_aicomm_bot
Q

API Key를 잃어버렸어요

⌄
Geminiaistudio.google.com → Get API key에서 재확인 또는 새 키 발급
Claudeconsole.anthropic.com → API Keys → 새 키 발급 (기존 키 재확인 불가)
GPTplatform.openai.com → API keys → 새 키 발급
DifyDify → 내 에이전트 → API 액세스에서 재확인 가능
Q

봇이 이전 대화를 기억 못해요 (매번 처음부터)

⌄
원인KV에 conversation_id가 저장 안 되거나, BOT_KV 바인딩 누락
해결 1Worker → Settings → Bindings에서 BOT_KV가 있는지 확인
해결 2코드에서 env.BOT_KV.put / env.BOT_KV.get이 제대로 작성됐는지 확인
