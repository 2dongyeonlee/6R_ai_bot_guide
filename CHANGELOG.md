# 개발 이력 & 패턴

> 이 봇을 만들면서 겪은 문제와 해결책. 새로 구현하는 사람이 같은 함정을 피하도록.

## 주요 아키텍처 결정

### Dify를 뺀 이유
- 처음엔 텔레그램 ↔ Dify ↔ LLM 구조였음.
- Dify는 RAG를 본격적으로 쓸 때 유용한데, 당시엔 RAG를 안 쓰고 있었음.
- 결과적으로 **외부 의존성만 추가 + 기밀이 미국 AWS 경유(보안 문제) + 속도 저하**.
- 제거하고 Worker가 Claude를 직접 호출 → **응답 약 2배 빨라짐**, 보안·관리 단순화.
- 향후 RAG가 필요하면 Cloudflare 자체 기능(Vectorize)으로 해결 가능.

### 서버 변천사
- Python + PythonAnywhere → 무료 서버들(Render/Railway/Koyeb) 시도 → 최종 Cloudflare Workers.
- 서버리스라 안 쓰면 0원, 자동 확장, 콜드스타트 없음.

### 모델 라우팅 (비용 절감)
- 단순 대화는 Haiku, 분석·브리핑은 Sonnet으로 자동 분기.
- `isComplexQuery`로 판단 (파일 있음 / 100자 초과 / 분석 키워드).

## 해결한 버그

### 1. 메시지 저장 누락
- **증상**: DM·단체방 메시지가 D1에 안 들어감.
- **원인**: ① 저장 코드가 라우팅 뒤에 있어 일부 경로에서 누락 ② sender_id가 NULL이면 INSERT가 조용히 실패.
- **해결**: handleMessage에서 **"저장 먼저, 라우팅 나중"** 순서 고정 + 모든 값 `String()` 캐스팅 + insertMessage 전체 try-catch + `console.log("saved ok:")`로 검증.

### 2. 날짜 검색 실패
- **증상**: "6/15 보고" 검색했는데 못 찾음.
- **원인**: created_at에 -2days 필터가 걸려 과거 자료 제외됨.
- **해결**: 날짜 필터 제거(전체 기간 검색) + `normalizeDateQuery`로 "오늘/내일/6월15일/6-15" → YYYY-MM-DD 변환해 milestone_date 매칭.

### 3. 외부 검색 안 됨
- **증상**: "검색해줘" 해도 "실시간 접근 불가"라고 답함.
- **원인**: 시스템 프롬프트에 "검색 불가" 문구 + Tavily 미연결.
- **해결**: searchWeb 함수 추가, "검색/찾아봐/최신" 키워드 트리거, 프롬프트에서 "검색 불가" 제거하고 "Tavily 연동됨" 명시.

### 4. 잡담까지 저장
- **증상**: "ㅎㅇ", "ㅋㅋ" 같은 것도 DB에 쌓임.
- **해결**: isJunk 필터 (3자 미만 / 자음모음만 / 특정 패턴 제외).

### 5. 봇이 없는 기능 수행한다고 답함
- **증상**: "OO한테 전해줘" 하면 실제론 안 하면서 "전달하겠습니다"라고만 함.
- **해결**: parseForwardRequest + handleForwardRequest 실제 구현. chat_id 없으면 "직접 연락 부탁".

## Codex/AI에게 코드 수정 지시할 때 (검증된 패턴)

효과적인 지시 형식:
```
[배경] 무슨 봇이고 지금 어떤 상황인지 1~2줄
[현재 문제] 구체적 증상
[기대 동작] 어떻게 작동해야 하는지
[검증 기준] 뭘 보면 고쳐진 걸 아는지
[절대 하지 말 것]
 - UTF-8 인코딩 변경 금지
 - 한글 문자열 직접 수정 금지
 - 파일 분리 금지
 - 불필요한 리팩토링 금지
 - 깨지면 git checkout worker.js로 복구
[완료 후] 배포하고 Version ID 알려줄 것
```

> "지시서"를 길게 쓰는 것보다 **"현재 문제 + 기대 동작 + 검증 기준"** 중심이 효과적.

### 한글 인코딩 주의
- PowerShell `(Get-Content)...Set-Content`로 한글 파일 직접 수정하면 **UTF-8이 깨짐**.
- 코드 수정은 에디터나 Codex로만. 깨지면 즉시 `git checkout worker.js`.
