01 — 구동 구조

봇이 작동하는 방식

텔레그램에 들어온 메시지·파일·이미지를 텍스트로 변환해 저장하고, 저장된 내용을 브리핑·검색·답변에 활용합니다.

1

수집

연결된 단체방·DM에 올라온 메시지, 파일, 이미지를 실시간으로 받습니다.

Telegram Webhook
2

저장

PDF·이미지를 읽어 텍스트로 바꾸고, 누가·언제·무슨 내용인지 요약해 데이터베이스에 저장합니다.

Claude + Cloudflare D1
3

검색

질문이 들어오면 키워드와 의미 두 방식으로 동시에 찾아 관련 자료를 선별합니다.

키워드 + 의미 검색 (Vectorize)
4

답변

찾은 내용을 정리해 답하거나, 원본 파일을 그대로 전송합니다. 아침 브리핑은 매일 9시에 자동 발송됩니다.

Claude + Telegram
Telegram → Cloudflare Workers → D1 · KV · Vectorize → Claude API
02 — 비서봇 기능

기능

질문 방식에 따라 자료를 파일로 전송하거나, 텍스트 내용으로 요약해 답합니다.

자료 검색·전송

여러 방에 올라온 파일·사진·보고 메시지를 찾아 전송합니다. 단체방 파일은 원본으로, DM 파일은 내용 텍스트로 제공합니다.

"OI 전략회의 자료 공유해줘"

아침 브리핑

주요 일정·보고·의사결정 사항을 분류해 매일 오전 9시에 자동 발송합니다. 오늘보다 과거 항목은 자동으로 제외됩니다.

📌 주요 일정 · 🔔 보고 · 💡 의사결정

사람별 내용 조회

이름을 대면 해당 인원이 공유한 보고·자료를 모아 요약합니다. 성·이름·영문명 모두 인식합니다.

"한혜승 담당이 공유한 내용 알려줘"

문서 내용 요약

파일에 답장(Reply) 후 요약을 요청하면 PDF·이미지·문서를 읽어 핵심만 정리합니다.

파일에 Reply → "요약해줘"

일정·의사결정 조회

날짜별 확정 일정과 판단이 필요한 사항을 분리해 알려줍니다.

"내일 일정 알려줘" / "오늘 의사결정 사항?"

외부 검색

사내 자료에 없는 내용은 실시간 웹 검색으로 보완하고 출처를 함께 제공합니다.

"최근 HBM 시장 동향 찾아줘"
03 — 운영 원칙

설계 기준

정확성과 보안을 위해 코드와 AI의 역할을 명확히 나눴습니다.

원칙 내용
날짜·분류는 코드가 결정 일정 분류·날짜 계산은 SQL로 처리. AI는 문장 생성만 담당합니다.
추측 없음 DB에 없는 내용은 만들지 않고 "확인 필요"로 안내합니다.
내부 처리 Dify 등 중간 도구 없이 Cloudflare 내에서만 처리합니다. 기밀 자료가 외부를 경유하지 않습니다.
단체방 파일 우선 같은 파일이 단체방·DM 양쪽에 있으면 단체방 버전을 먼저 전송합니다.
DM으로만 공유된 파일은 file_id가 봇 전용이 아니어서 재전송이 불가합니다. 중요 자료는 OHKwon_bot이 참여한 단체방에 올리면 이후 요청 시 원본 파일로 전송됩니다.

05 — 전체 구조

한눈에 보는 구조

들어온 모든 것(텍스트·파일·이미지)을 텍스트화해 D1에 저장하고, 저장된 자료를 브리핑·검색·답변이 재사용합니다. Dify 같은 중간 도구 없이 Worker가 Claude를 직접 호출합니다 — 빠르고, 기밀이 외부를 경유하지 않습니다.

[텔레그램]
    ↓ 메시지·파일 수신
[Cloudflare Workers]   ← 봇 로직 (worker.js)
    ↓
[Cloudflare D1]        ← 모든 메시지·요약·파일 정보 저장
[Cloudflare KV]        ← 시스템 프롬프트·대화 히스토리
[Cloudflare Vectorize] ← 의미 기반 검색 색인
    ↓
[Claude API]           ← 요약·분석·답변 (두뇌)
[Tavily API]           ← 실시간 외부 검색 (선택)
    ↓
[텔레그램]             ← 답변 전송

07 — 단계별 구축

봇 만들기 8단계

1. 텔레그램 봇 생성

텔레그램에서 @BotFather → /newbot → 이름·사용자명(_bot으로 끝) 설정 → 봇 토큰 저장. /setprivacy → Disable (그룹방 전체 메시지 수신에 필수).

2. Cloudflare 세팅

npm install -g wrangler
wrangler login

3. D1 데이터베이스 생성

wrangler d1 create 6r-ai-db
# 출력되는 database_id 복사
wrangler d1 execute 6r-ai-db --remote --file=schema.sql

messages 테이블(room_id·sender_id·content·status_tag·milestone_date·file_id·summary 등)을 schema.sql로 생성합니다.

4. KV 네임스페이스 생성

wrangler kv namespace create PROMPT   # 시스템 프롬프트·대화 히스토리

5. wrangler.toml 작성

name·main·compatibility 설정과 [vars](BRIEFING_CHAT_ID·BOT_USERNAME), [triggers] crons = ["0 0 * * 1-5"](KST 09:00 월~금), [[d1_databases]]·[[kv_namespaces]] 바인딩을 채웁니다.

6. 비밀 키 등록

wrangler secret put TELEGRAM_BOT_TOKEN
wrangler secret put ANTHROPIC_API_KEY
wrangler secret put TAVILY_API_KEY   # (선택)

7. 배포

wrangler deploy   # 출력되는 Worker URL 복사

8. 텔레그램 웹훅 연결

curl "https://api.telegram.org/bot<봇토큰>/setWebhook?url=<WorkerURL>"

{"ok":true}가 나오면 완료. 봇에게 메시지를 보내면 응답합니다.

08 — 핵심 코드 구조

worker.js 흐름

처리 순서가 중요합니다. 저장이 먼저, 라우팅이 나중입니다. (과거 저장이 라우팅 뒤에 있어 메시지가 누락된 버그가 있었음)

1. 봇 자신 메시지면 무시
2. /설정 명령 → 시스템 프롬프트 수정
3. 파일·이미지 → 추출·저장·요약 (ingestAndSummarize)
4. 텍스트 → 잡담 아니면 먼저 DB 저장 (saveMessage)
5. #보고 등 태그 → 업무보고 처리 (handleReport)
6. 봇 멘션·질문 → 답변 생성 (handleQuery)
7. 그 외는 저장만 (응답 없음)

모델 라우팅 (비용 절감 핵심)

단순 대화는 Haiku, 분석·브리핑은 Sonnet으로 자동 분기합니다.

const MODEL_FAST  = "claude-haiku-4-5-20251001";  // 단순 대화
const MODEL_SMART = "claude-sonnet-4-6";          // 분석·브리핑

function isComplexQuery(query, hasFile) {
  if (hasFile) return true;
  if (query.length > 100) return true;
  return /(요약|분석|보고|브리핑|정리|검토|작성|전략|판단|비교|예측)/.test(query);
}
09 — 파일 처리 & 검색

파일 처리

형식처리 방식
PDFClaude에 base64 document로 직접 전달 (표·레이아웃까지 인식)
이미지Claude Vision으로 텍스트화 (describeImage)
PPTX/DOCX/XLSX파일에 Reply 후 요약 요청 시 텍스트 추출해 답변

자료 검색 (하이브리드)

키워드와 의미 검색을 동시에 돌려 병합·중복제거 후 상위 12개를 반환합니다.

  • 키워드 검색 — content·summary·file_name·sender_name LIKE
  • 의미 검색(Vectorize) — 질문을 임베딩해 의미 유사 항목 조회
  • 발신자 명시 검색 — "예슬TL이 보고한 자료"처럼 사람이 명시되면 해당 인원 우선 (NAME_ALIASES로 이름·성·영문명 변형 매칭)
  • 자료 통합 — 단체방 파일은 copyMessage로 원본 전송, DM 전용 파일은 텍스트로 제공, 같은 파일이 양쪽에 있으면 단체방 버전 우선
내부 자료에 없거나 "검색해줘/최신" 요청이면 Tavily 외부 검색으로 보완하고 출처 URL을 함께 제공합니다.
10 — 브리핑 & 프롬프트

아침 브리핑

cron(KST 09:00 월~금)에 자동 실행. 날짜·태그 분류는 코드(SQL)가 결정적으로 처리하고 문장만 Claude가 생성합니다.

📅 YYYY-MM-DD Daily Briefing
📌 주요 일정   (임원·담당급 언급, 날짜 확정 일정)
🔔 보고        (팀장·TL 진행 보고)
💡 의사결정 필요
📋 주요 내용   (방별 핵심 공유)

오늘보다 과거 날짜 항목은 코드 필터로 자동 제외됩니다.

시스템 프롬프트 규칙

KV에 저장되어 /설정 명령으로 재배포 없이 수정 가능합니다.

  • 존댓말, 짧고 간결하게, 바로 본론
  • 마크다운(**,#,*) 금지. HTML <b> 태그만 사용
  • 문장 끝 "~입니다" 금지, 명사형으로 마무리
  • 블릿(•) 사용 (모바일 가독성)
  • 추측 금지 — 없는 정보는 만들지 않음
