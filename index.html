Overview
텔레그램 AI 비서봇 — 기능 & 동작 흐름
각 구성원이 텔레그램에서 자기 전담 AI 비서봇과 대화한다. 들어온 메시지·파일을 저장하고, 물어보면 쌓인 자료에서 찾아 답하며, 매일 아침 브리핑을 보낸다. Dify 같은 중간 도구 없이 Cloudflare Worker가 Claude를 직접 호출한다.
⭐ 비서봇이 하는 일
📂
자료 검색·전송 — "OO 자료 공유해줘" → 단체방 원본 파일 또는 텍스트로 제공
📅
아침 브리핑 — 평일 아침 주요 일정·보고·의사결정 자동 발송
🙋
사람별 조회 — "예슬TL이 보고한 내용" → 발신자 기준 모아 요약
📄
문서 요약 — 파일에 Reply 후 "요약해줘" → PDF·이미지·Office 추출
🌐
외부 검색 — 사내 자료에 없으면 Tavily 웹 검색으로 보완
↪️
메시지 전달 — "OO에게 전해줘" → 해당 인원 DM으로 전송
🔄 메시지 처리 전체 흐름
📱 텔레그램에 메시지·파일·이미지 도착 ↓ (Webhook — 텔레그램이 Cloudflare로 자동 전달) ⚡ Cloudflare Worker (worker.js) ├── 저장 먼저 → 🗄️ D1 저장 + 🧷 Vectorize 색인 (bge-m3 임베딩) └── 질문이면 → 🔎 하이브리드 검색 (키워드 + 의미) ↓ 🧠 Claude 호출 api.anthropic.com (단순=Haiku / 분석=Sonnet) ↓ 📱 텔레그램으로 답변·파일 전송
🧩
핵심 4요소 — D1(저장) · Vectorize+Workers AI(의미 검색) · KV(프롬프트·히스토리) · Claude API(두뇌). 각각은 Step 3·4에서 만든다.
💡 구성 요소 한눈에
도구역할 (worker.js)바인딩/키비용
📱 텔레그램채팅 창구 + 파일 전송 (BotFather)TELEGRAM_BOT_TOKEN무료
⚡ Workers봇 로직 worker.js · 24시간 서버리스—무료
🗄️ D1messages 테이블 (메시지·요약·파일)env.DB무료
📦 KV시스템 프롬프트·대화 히스토리env.PROMPT무료
🧷 Vectorize의미 검색 색인 (koh-ai-vectors)env.VECTORIZE무료
🤖 Workers AI임베딩 생성 (@cf/baai/bge-m3)env.AI무료

🔍 Tavily실시간 외부 검색 (선택)TAVILY_API_KEY월 1,000건 무료
Feature
📂 자료 검색·전송
여러 방에 올라온 파일·사진·보고 메시지를 찾아 전송한다. 질문 방식에 따라 원본 파일로 보내거나 텍스트로 요약해 답한다.
💬 이렇게 쓴다
🗣️
"OI 전략회의 자료 공유해줘" · "6/15 보고 뭐 있어?" · "한혜승 담당이 공유한 내용 알려줘"
상황봇의 응답 (worker.js)
단체방 파일copyMessage로 원본 파일 그대로 전송
단체방·DM 양쪽에 있는 파일단체방 버전 우선 전송
DM 전용 파일sendDocument(file_id) 또는 저장된 내용 텍스트로 제공
텍스트 보고"누가·무슨 내용" 형태로 요약 답변
⚠️
DM으로만 공유된 파일은 file_id가 봇 전용이 아니어서 재전송이 불가. 중요 자료는 봇이 참여한 단체방에 올리면 이후 원본 파일로 전송된다.
Feature
📅 아침 브리핑
cron(0 0 * * 1-5 = UTC 00:00 = KST 09:00, 평일)에 runMorningBriefing이 자동 실행. 주요 일정·보고·의사결정을 분류해 BRIEFING_CHAT_ID 방에 발송한다.
📋 브리핑 양식
📅 YYYY-MM-DD Daily Briefing 📌 주요 일정 (임원·담당급 언급, 날짜 확정 일정) 🔔 보고 (팀장·TL 진행 보고) 💡 의사결정 필요 📋 주요 내용 (방별 핵심 공유)
🎯
날짜·태그 분류는 코드(SQL)가 결정적으로 처리하고 문장만 Claude가 생성 — 날짜 계산이 틀리지 않는다. 오늘보다 과거 날짜 항목은 자동 제외.
Feature
🧩 조회 · 요약 · 외부 검색
사람별 조회, 문서 요약, 일정·의사결정 조회, 외부 웹 검색, 메시지 전달까지 — 일상 업무 보조 기능들.
🙋 사람별 내용 조회

searchBySender — 이름을 대면 해당 인원이 공유한 보고·자료를 모아 요약. buildNameAliases로 이름·성·영문명 변형 자동 인식.

🗣️
"한혜승 담당이 공유한 내용 알려줘"
📄 문서 내용 요약

파일에 답장(Reply) 후 요약 요청 → PDF(unpdf)·이미지(Vision)·Office(extractOfficeText) 추출해 핵심 정리.

🗣️
파일에 Reply → "요약해줘"
🗓️ 일정·의사결정 조회

handleScheduleQuery — 날짜별 확정 일정과 판단이 필요한 사항을 분리해 알려줌.

🗣️
"내일 일정 알려줘" · "오늘 의사결정 사항?"
🌐 외부 검색 · ↪️ 전달

searchWeb(Tavily)로 외부 보완 + 출처 제공. handleForwardRequest로 "OO에게 전해줘" → 해당 인원 DM 전송.

🗣️
"최근 HBM 동향 찾아줘" · "권오혁 담당께 전달해줘"
Step 1 / 4
📱 텔레그램 봇 생성
BotFather에서 봇 이름을 만들고 Token(봇의 열쇠)을 받는다. 약 5분.
🔑
Bot Token이란? 텔레그램이 "이게 진짜 네 봇"임을 확인하는 비밀 문자열. 나중에 Cloudflare secret TELEGRAM_BOT_TOKEN에 입력한다. 절대 공유 금지.

봇 토큰을 안전하게 저장했으면 완료.
Step 2 / 4
🧠 LLM 연동 (Claude)
봇의 두뇌인 Claude API 키를 발급받는다. worker.js는 Anthropic Messages API를 직접 호출한다 — Dify·중간 서버 없음. 약 5분.
🧠
worker.js는 https://api.anthropic.com/v1/messages를 직접 호출한다. 단순 대화는 Haiku, 분석·브리핑은 Sonnet으로 자동 분기(비용 절감). 자세한 호출 구조는 좌측 LLM 호출 (callClaude) 참고.

  room_id TEXT NOT NULL,
  room_title TEXT,
  sender_id TEXT NOT NULL,
  sender_name TEXT,
  content TEXT NOT NULL,
  status_tag TEXT DEFAULT '',
  field_tag TEXT DEFAULT '',
  milestone_date TEXT DEFAULT '',
  file_id TEXT DEFAULT '',
  file_name TEXT DEFAULT '',
  summary TEXT DEFAULT '',
  action_items TEXT DEFAULT '',
  needs_escalation INTEGER DEFAULT 0,
  info_tag TEXT DEFAULT '',

KV · Vectorize 생성

프롬프트 저장 + 의미 검색 색인

⌄
터미널

# Vectorize 색인 (bge-m3 = 1024차원, 코사인)

💡

wrangler.toml 작성

D1·KV·Vectorize·AI 바인딩 + cron

⌄

name = "내봇이름"
main = "worker.js"
compatibility_date = "2024-11-01"
compatibility_flags = ["nodejs_compat"]
keep_vars = true

[vars]

[triggers]
crons = ["0 0 * * 1-5"]   # UTC 00:00 = KST 09:00, 평일

[[d1_databases]]
binding = "DB"
database_name = "6r-ai-db"
database_id = "위에서_복사한_DB_ID"

[[kv_namespaces]]
binding = "PROMPT"
id = "위에서_복사한_KV_ID"
preview_id = "위에서_복사한_KV_ID"

[[vectorize]]
binding = "VECTORIZE"
index_name = "koh-ai-vectors"

[ai]
binding = "AI"

Internals
🔌 LLM 호출 (callClaude)
worker.js는 Anthropic Messages API를 직접 호출한다. 시스템 프롬프트는 KV에서 불러오고, 질문 복잡도에 따라 모델을 자동 선택한다.
📡 Anthropic API 직접 호출
worker.js · callClaude()
const MODEL_FAST  = "claude-haiku-4-5-20251001";  // 단순 대화
const MODEL_SMART = "claude-sonnet-4-6";          // 분석·브리핑

const res = await fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: {
    "content-type": "application/json",
    "x-api-key": env.ANTHROPIC_API_KEY,
    "anthropic-version": "2023-06-01",
  },
  body: JSON.stringify({
    model,                                   // Fast 또는 Smart
    max_tokens: model === MODEL_FAST ? 500 : 1500,
    system: effectiveSystem,                 // KV의 시스템 프롬프트
    messages: [{ role: "user", content: userText }],
  }),
});
🖼️
이미지는 describeImage()가 같은 엔드포인트에 base64 image 블록으로 보내 Claude Vision으로 텍스트화한다. PDF는 base64 document로 직접 전달.
⚖️ 모델 자동 분기 (isComplexQuery)
worker.js · isComplexQuery()
function isComplexQuery(query, hasFile) {
  if (hasFile) return true;
  if (query.length > 100) return true;
  return /(요약|분석|보고|브리핑|정리|검토|작성|전략|판단|비교|예측)/.test(query);
}
💸
파일이 있거나 100자 초과, 분석 키워드면 Sonnet(똑똑·비쌈), 그 외엔 Haiku(빠름·저렴). 품질은 유지하고 비용은 절감.
Internals
🧭 코드 구조 & 라우팅
handleMessage의 핵심은 처리 순서다. 저장이 먼저, 라우팅이 나중 — 과거 저장이 라우팅 뒤에 있어 메시지가 누락된 버그가 있었다.
📨 메시지 라우팅 순서
1. 봇 자신 메시지면 무시 2. /설정 · /reindex 등 명령 처리 3. 파일·이미지 → 추출·저장·요약 (ingestAndSummarize) 4. 텍스트 → 잡담(isJunk) 아니면 먼저 DB 저장 (saveMessage) 5. 보고 태그 → 업무보고 처리 (handleReport) 6. 봇 멘션·DM 질문 → 답변 생성 (handleQuery) 7. 그 외는 저장만 (응답 없음)
🧠 질문 처리 흐름 (handleQuery)
1
대화 히스토리 로드 (getChatHistory · KV)
2
내부 자료 검색 (searchMemory → D1 + Vectorize)
3
내부에 없으면 외부 검색 (searchWeb → Tavily)
4
복잡도 판단(isComplexQuery) → 모델 선택 → callClaude
5
히스토리 저장 (saveChatHistory) · 파일 요청이면 재전송
Internals
🔎 자료 검색 원리 (하이브리드)
searchMemory가 키워드 검색과 의미 검색을 동시에 돌려 병합·중복제거한다.
🧬 하이브리드 검색
1
키워드 검색(likeSearch) — content·summary·file_name·sender_name LIKE
2
