01 — 구동 구조

봇이 작동하는 방식

텔레그램에 들어온 메시지·파일·이미지를 텍스트로 변환해 저장하고, 저장된 내용을 브리핑·검색·답변에 활용합니다.

1

수집

연결된 단체방·DM에 올라온 메시지, 파일, 이미지를 실시간으로 받습니다.

Telegram Webhook
2

저장

PDF·이미지를 읽어 텍스트로 바꾸고, 누가·언제·무슨 내용인지 요약해 데이터베이스에 저장합니다.

Claude + Cloudflare D1
3

검색

질문이 들어오면 키워드와 의미 두 방식으로 동시에 찾아 관련 자료를 선별합니다.

키워드 + 의미 검색 (Vectorize)
4

답변

찾은 내용을 정리해 답하거나, 원본 파일을 그대로 전송합니다. 아침 브리핑은 매일 9시에 자동 발송됩니다.

Claude + Telegram
Telegram → Cloudflare Workers → D1 · KV · Vectorize → Claude API
02 — 비서봇 기능

기능

질문 방식에 따라 자료를 파일로 전송하거나, 텍스트 내용으로 요약해 답합니다.

자료 검색·전송

여러 방에 올라온 파일·사진·보고 메시지를 찾아 전송합니다. 단체방 파일은 원본으로, DM 파일은 내용 텍스트로 제공합니다.

"OI 전략회의 자료 공유해줘"

아침 브리핑

주요 일정·보고·의사결정 사항을 분류해 매일 오전 9시에 자동 발송합니다. 오늘보다 과거 항목은 자동으로 제외됩니다.

📌 주요 일정 · 🔔 보고 · 💡 의사결정

사람별 내용 조회

이름을 대면 해당 인원이 공유한 보고·자료를 모아 요약합니다. 성·이름·영문명 모두 인식합니다.

"한혜승 담당이 공유한 내용 알려줘"

문서 내용 요약

파일에 답장(Reply) 후 요약을 요청하면 PDF·이미지·문서를 읽어 핵심만 정리합니다.

파일에 Reply → "요약해줘"

일정·의사결정 조회

날짜별 확정 일정과 판단이 필요한 사항을 분리해 알려줍니다.

"내일 일정 알려줘" / "오늘 의사결정 사항?"

외부 검색

사내 자료에 없는 내용은 실시간 웹 검색으로 보완하고 출처를 함께 제공합니다.

"최근 HBM 시장 동향 찾아줘"
03 — 운영 원칙

설계 기준

정확성과 보안을 위해 코드와 AI의 역할을 명확히 나눴습니다.

원칙 내용
날짜·분류는 코드가 결정 일정 분류·날짜 계산은 SQL로 처리. AI는 문장 생성만 담당합니다.
추측 없음 DB에 없는 내용은 만들지 않고 "확인 필요"로 안내합니다.
내부 처리 Dify 등 중간 도구 없이 Cloudflare 내에서만 처리합니다. 기밀 자료가 외부를 경유하지 않습니다.
단체방 파일 우선 같은 파일이 단체방·DM 양쪽에 있으면 단체방 버전을 먼저 전송합니다.
DM으로만 공유된 파일은 file_id가 봇 전용이 아니어서 재전송이 불가합니다. 중요 자료는 OHKwon_bot이 참여한 단체방에 올리면 이후 요청 시 원본 파일로 전송됩니다.
