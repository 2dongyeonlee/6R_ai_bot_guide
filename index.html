Overview
텔레그램 AI 비서봇 — 기능 & 동작 흐름
각 구성원이 텔레그램에서 자기 전담 AI 비서봇과 대화한다. 들어온 메시지·파일을 저장하고, 물어보면 쌓인 자료에서 찾아 답하며, 매일 아침 브리핑을 보낸다. Dify 같은 중간 도구 없이 Cloudflare가 Claude를 직접 호출한다.
⭐ 비서봇이 하는 일
📂
자료 검색·전송 — "OO 자료 공유해줘" → 단체방 원본 파일 또는 텍스트로 제공
📅
아침 브리핑 — 평일 아침 주요 일정·보고·의사결정 자동 발송
🙋
사람별 조회 — "000이 보고한 내용" → 발신자 기준 모아 요약
📄
문서 요약 — 파일에 답장 후 "요약해줘" → PDF·이미지·문서 추출
🌐
외부 검색 — 사내 자료에 없으면 실시간 웹 검색으로 보완
↪️
메시지 전달 — "000에게 전해줘" → 해당 인원에게 전송
🔄 메시지 처리 전체 흐름
📱 텔레그램에 메시지·파일·이미지 도착 ↓ (텔레그램이 메시지를 봇 서버로 자동 전달) ⚡ Cloudflare (봇 서버) ├── 먼저 저장 → 🗄️ 데이터베이스에 저장 └── 질문이면 → 🔎 똑똑한 검색 (단어 + 뜻) ↓ 🧠 Claude 호출 단순=빠른 모델 / 분석=똑똑한 모델 ↓ 📱 텔레그램으로 답변·파일 전송
🧩
봇을 이루는 구성 요소는 좌측 구성 요소 페이지에서, 만드는 방법은 구축 단계에서 확인.
⏱ 구축 단계 한눈에 — 처음부터 끝까지 약 30분
단계하는 일소요

Step 2두뇌(Claude) API 키 발급약 5분

Features

📂 자료 검색·전송
🗣️
"OI 전략회의 자료 공유해줘" · "6/15 보고 뭐 있어?"
상황봇의 응답
단체방 파일원본 파일을 그대로 전송

🙋 사람별 조회 · 📄 문서 요약 · 🌐 외부 검색 · ↪️ 전달

📄 문서 요약파일에 답장(Reply) 후 "요약해줘" → PDF·이미지·문서 읽어 핵심 정리
🗓️ 일정·의사결정"내일 일정 알려줘" · "오늘 의사결정 사항?"

Step 1 / 4 · ⏱ 약 5분
📱 텔레그램 봇 만들기
텔레그램 안에서 "봇을 만들어주는 봇(BotFather)"과 대화해 내 봇을 만들고, 봇의 열쇠(토큰)를 받는다. 약 5분.
📍
어디서: 텔레그램 앱 안에서 진행. 컴퓨터·휴대폰 어디든 OK.
1

BotFather와 대화해 봇 생성

이름 정하고 토큰 받기

⌄
1
텔레그램 검색창에 @BotFather 입력 → 파란 체크(✓) 공식 계정 선택
2
대화창에 /newbot 보내기
3
봇 이름과 사용자명(끝이 _bot) 입력 (안내대로 따라가면 됨)
4
BotFather가 주는 토큰을 복사해 메모장에 저장 (예: 1234567:ABC-xyz...) — 비밀번호처럼 다룰 것
2

그룹방 메시지 다 받게 설정

프라이버시 끄기

⌄
1
BotFather에 /setprivacy 보내기 → 방금 만든 봇 선택
2
Disable 선택
⚠️
이걸 안 하면 그룹방에서 봇을 직접 부른 메시지만 받는다. 방의 모든 대화를 저장하려면 꼭 Disable.
토큰을 메모장에 저장했으면 완료.
Step 2 / 4 · ⏱ 약 5분
🧠 봇의 두뇌(Claude) 연결
봇이 생각하고 답하려면 Claude를 쓸 "API 키"가 필요하다. 키를 발급받아 메모해두고, 실제 등록은 Step 4에서 한다. 약 5분.
📍
어디서: Anthropic 사이트(console.anthropic.com)에서 가입하고 키를 만든다.
1

Claude API 키 발급

console.anthropic.com

⌄
1
console.anthropic.com 가입 → 결제 카드 등록 (사용량만큼 과금)
2
왼쪽 메뉴 API Keys → Create Key 클릭
3
만들어진 키를 복사해 메모장에 저장 (한 번만 보여줌, 비밀번호처럼 다룰 것)
2

(선택) 외부 웹 검색 키

실시간 검색이 필요하면

⌄
1
tavily.com 가입 → API Key 발급 (월 1,000건 무료)
2
키를 메모장에 저장 (이것도 Step 4에서 등록). 안 해도 외부 검색만 빠지고 나머지는 정상 동작
🤖
"뜻으로 찾는 검색"에 쓰는 AI는 Cloudflare가 무료로 처리하므로 별도 키가 필요 없다.
Claude 키를 메모장에 저장했으면 완료.
Step 3 / 4 · ⏱ 약 10분
⚡ Cloudflare에 저장소 만들기
봇이 쓸 저장 공간(데이터베이스·검색 색인 등)을 만든다. 여기서부터는 '터미널'에 명령어를 붙여넣는다. 약 10분.
💻
터미널이 뭔가요? 컴퓨터에 명령어를 입력하는 검은 창. Windows는 시작 메뉴에서 "PowerShell", Mac은 "터미널" 앱을 열면 된다. 아래 회색 박스의 복사를 눌러 붙여넣고 Enter만 누르면 끝.
1

도구 설치 & 로그인

딱 한 번만

⌄

터미널에 아래를 붙여넣기. 두 번째 줄을 실행하면 브라우저가 열리며 Cloudflare 로그인 → 허용.

터미널에 붙여넣기
npm install -g wrangler

2

데이터베이스(D1) 만들기

메시지·파일을 저장할 곳

⌄

아래를 붙여넣어 실행하면 database_id가 화면에 뜬다. 그 값을 메모장에 저장(Step 4에서 사용).

터미널에 붙여넣기
wrangler d1 create 6r-ai-db

이어서, 저장할 표(테이블)를 만든다. 같은 폴더에 schema.sql 파일을 만들어 아래 내용을 붙여넣고 저장한 뒤, 그 아래 명령을 실행.

schema.sql 에 붙여넣기
CREATE TABLE IF NOT EXISTS messages (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  telegram_message_id TEXT,
  room_id TEXT NOT NULL,
  room_title TEXT,
  sender_id TEXT NOT NULL,
  sender_name TEXT,
  content TEXT NOT NULL,
  status_tag TEXT DEFAULT '',
  field_tag TEXT DEFAULT '',
  milestone_date TEXT DEFAULT '',
  file_id TEXT DEFAULT '',
  file_name TEXT DEFAULT '',
  summary TEXT DEFAULT '',
  action_items TEXT DEFAULT '',
  needs_escalation INTEGER DEFAULT 0,
  info_tag TEXT DEFAULT '',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
터미널에 붙여넣기
wrangler d1 execute 6r-ai-db --remote --file=schema.sql
3

말투 저장소 · 똑똑한 검색 만들기

한 번에 두 줄 실행

⌄

아래 두 줄을 붙여넣어 실행. 첫 줄 실행 후 나오는 id를 메모장에 저장(Step 4에서 사용). 둘째 줄은 "단어가 달라도 뜻으로 찾는 검색"을 준비하는 명령 — 그대로 붙여넣으면 된다.

터미널에 붙여넣기
wrangler kv namespace create PROMPT
wrangler vectorize create ai-bot-search --dimensions=1024 --metric=cosine
🤖
검색용 AI는 따로 만들 필요 없다. Step 4의 설정 파일에 두 줄만 넣으면 켜진다.
database_id · KV id를 메모장에 저장했으면 완료.
Step 4 / 4 · ⏱ 약 10분
🚀 설정 채우고 봇 띄우기
지금까지 메모한 값들을 설정 파일에 넣고, 키를 등록한 뒤 봇을 배포한다. 약 10분.
1

설정 파일(wrangler.toml) 채우기

메모해둔 id를 넣는 곳

⌄

봇 폴더의 wrangler.toml 파일에 아래를 붙여넣고, 대괄호 표시 부분을 메모한 값으로 교체.

wrangler.toml 에 붙여넣기
name = "내봇이름"
main = "worker.js"
compatibility_date = "2024-11-01"
compatibility_flags = ["nodejs_compat"]
keep_vars = true

[vars]
BRIEFING_CHAT_ID = "-방ID1,-방ID2"   # 아침 브리핑 보낼 방
BOT_USERNAME = "내봇이름_bot"          # @ 제외

[triggers]
crons = ["0 0 * * 1-5"]   # 평일 아침 9시(한국)

[[d1_databases]]
binding = "DB"
database_name = "6r-ai-db"
database_id = "[메모한_DB_ID]"

[[kv_namespaces]]
binding = "PROMPT"
id = "[메모한_KV_ID]"
preview_id = "[메모한_KV_ID]"

[[vectorize]]
binding = "VECTORIZE"
index_name = "ai-bot-search"

[ai]
binding = "AI"

🔌 LLM 호출 방식
봇은 Claude를 직접 호출한다(중간 도구 없음). 시스템 프롬프트는 따로 저장된 걸 불러오고, 질문이 단순한지 복잡한지에 따라 모델을 자동으로 바꾼다.
📡 Claude 직접 호출
callClaude()
const MODEL_FAST  = "claude-haiku-4-5-20251001";  // 단순 대화 (빠름·저렴)
const MODEL_SMART = "claude-sonnet-4-6";          // 분석·브리핑 (똑똑함)

const res = await fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: {
    "content-type": "application/json",
    "x-api-key": env.ANTHROPIC_API_KEY,
    "anthropic-version": "2023-06-01",
  },
  body: JSON.stringify({
    model,                                   // 빠른 모델 또는 똑똑한 모델
    max_tokens: model === MODEL_FAST ? 500 : 1500,
    system: effectiveSystem,                 // 저장된 시스템 프롬프트
    messages: [{ role: "user", content: userText }],
  }),
});
🖼️
이미지는 같은 방식으로 보내 Claude가 내용을 읽어 텍스트로 바꾼다. PDF도 그대로 전달해 표·서식까지 인식.
⚖️ 모델 자동 선택
isComplexQuery()
function isComplexQuery(query, hasFile) {
  if (hasFile) return true;
  if (query.length > 100) return true;
  return /(요약|분석|보고|브리핑|정리|검토|작성|전략|판단|비교|예측)/.test(query);
}
💸
파일이 있거나 100자 초과, 분석 키워드면 똑똑한 모델, 그 외엔 빠른 모델. 품질은 유지하고 비용은 절감.

🧭 코드 구조 & 라우팅
봇은 메시지가 들어오면 먼저 저장하고, 그다음 무엇을 할지 판단한다. (예전엔 순서가 반대라 메시지가 누락된 적이 있었다.)
📨 메시지가 들어오면 봇이 하는 일 (순서대로)
1
봇이 보낸 메시지면 그냥 넘어감 — 자기 답에 또 답하는 무한 반복 방지
2
명령어면 명령 처리 — 아래 "봇 명령어" 참고
3
파일·이미지면 내용을 읽어 요약하고 저장
4
일반 텍스트면 의미 없는 잡담("ㅋㅋ" 등)이 아니면 먼저 저장
5
봇을 부르거나 1:1 질문이면 답변 생성 (아래 흐름)
6
그 외에는 저장만 하고 답하지 않음 (단체방 도배 방지)
⌨️ 봇 명령어 (관리자가 텔레그램에서 입력)
명령어무슨 뜻 / 언제 쓰나
/설정 ...봇의 말투·규칙을 바꿈. 예: /설정 일정은 표로 정리해줘 → 코드 수정·재배포 없이 즉시 반영
/reindex예전에 쌓인 대화·파일을 "뜻으로 찾는 검색"에 넣어줌. 봇을 처음 켰을 때 한 번, 자료가 많으면 몇 번 반복
💡
명령어는 봇과의 텔레그램 대화창에 그냥 메시지처럼 입력하면 된다.
🧠 질문에 답하는 흐름
1
최근 대화 맥락 불러오기
2
내부 자료 검색 (키워드 + 의미)
3
내부에 없으면 외부 웹 검색
4
질문 복잡도 판단 → 모델 선택 → Claude 호출
5
대화 저장 · 파일 요청이면 파일 재전송

🔎 자료 검색 방식
키워드 검색과 의미 검색을 동시에 돌려 합친 뒤, 중복을 빼고 가장 관련 있는 것부터 보여준다.
🧬 단어 + 뜻, 두 가지로 찾기
1
단어로 찾기 — 내용·요약·파일명·보낸사람에서 같은 단어 검색
2
뜻으로 찾기 — 단어가 달라도 의미가 비슷한 자료를 찾아줌
3
두 결과를 합치고 중복 제거 후 관련도 순 정렬
✨
키워드만 쓰던 때는 "M15X 사고"를 "화재"로 못 찾았다. 의미 검색이 붙으며 이 한계가 풀렸다. 예전 자료는 봇에 /reindex를 보내 색인.
📎 파일 읽기
형식처리
PDF텍스트 추출 + Claude에 그대로 전달 (표·서식 인식)
이미지Claude가 보고 내용을 텍스트로 변환
PPTX/DOCX/XLSX파일에 답장 후 요약 요청 시 텍스트 추출

💬 시스템 프롬프트 규칙
봇의 말투·답변 품질을 좌우하는 핵심. 텔레그램에서 /설정 ... 명령으로 재배포 없이 바로 바꿀 수 있다.
✅ 검증된 규칙
1
존댓말, 짧고 간결하게, 바로 본론
2
마크다운(**,#,*) 금지. HTML <b> 태그만 사용
3
문장 끝 "~입니다" 금지. 명사형으로 끝낼 것
4
블릿포인트(•) 사용 (모바일 가독성)
5
긴급(오늘·D-1)은 🚨, D-7 이내는 ⚠️ 표시
6
추측 금지. 없는 정보는 만들지 않음
⚙️
텔레그램에서 /설정 앞으로 일정은 표로 정리처럼 보내면 지시가 누적되어 즉시 반영된다.
